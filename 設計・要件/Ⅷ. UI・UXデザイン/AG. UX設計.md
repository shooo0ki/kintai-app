# AG. UX設計

## 概要

UX設計は、ユーザーが目的を達成するまでの全体の流れを設計するプロセスです。心理、行動パターン、タスクの流れを理解し、ストレスの少ない体験を実現します。

---

## 1. ユーザーフロー図

**目的:** ユーザーの行動分岐を可視化

**要素:**
- 開始地点・終了地点
- 各ステップの判断分岐（Yes/No）
- 離脱リスクが高い地点の特定

**分析項目:**
- 心理状態の変化（期待感 → 判断 → 決定 → 安心）
- 痛点（複雑なフロー、予想外の価格等）
- 離脱地点（3ステップ以上の操作が必要 → 離脱のリスク）

---

## 2. 情報アーキテクチャ（IA）

**目的:** 情報をどのように分類・整理・ナビゲートするか決める

**要素:**
1. **組織化** - 階層型（ツリー）/ マトリックス型（複数視点）/ タグベース
2. **ナビゲーション** - グローバルナビ、ローカルナビ、パンくずリスト
3. **検索機能** - キーワード検索、フィルター、オートコンプリート
4. **メタデータ** - ページタイトル、説明文、カテゴリタグ

**設計プロセス:**
1. ユーザー・ビジネス要件を分析
2. カテゴリ分けを複数パターン検討
3. ツリー図として整理
4. カードソーティング・ツリーテストで検証

---

## 3. タスク分析

**目的:** ユーザーの大きなゴールを小さなステップに分解

**構造例:**
```
メインゴール：オンラインで衣類を購入
├─ 準備段階：何が欲しいかイメージ作成
│  └─ 季節・用途・予算を決める
├─ 発見段階：候補を検索・探索
│  └─ 3ステップ以上の操作が必要 → 改善必要
├─ 評価段階：商品詳細を確認
├─ 決定段階：カートに追加
└─ 実行段階：決済・注文完了
```

**痛点と改善:**
- トップページに「オススメ検索」を表示（直接提案）
- ワンステップで複数条件を同時入力（カテゴリ・価格・色）
- 検索結果をリアルタイム更新（フィルター変更時）

---

## 4. インタラクションデザイン

**原則:** ユーザーの期待と現実が一致すること

**良い例：「カートに追加」ボタンをクリック**
- ボタンが視覚的に「押された」状態に変化
- ローディング表示（処理中を示す）
- カート数が更新される
- トースト通知で「カートに追加しました」と表示

**実装時の状態管理:**
```
idle（アイドル）
  ↓ [クリック]
loading（ローディング）- スピナー表示
  ↓ [成功]
success（成功）- チェックマーク表示
  ↓ [2秒後]
idle（アイドル）に戻る
```

---

## 5. UX設計の5つの原則

1. **ユーザー中心** - デザイナー想定ではなく、ユーザー行動を基準に
2. **明確性** - 「今何ができるのか」「何が起こるのか」を明確に
3. **一貫性** - ボタンはいつも同じ場所、同じ見た目
4. **フィードバック** - ユーザーの操作に応答を返す
5. **効率性** - よく使う操作は1ステップ、複雑な操作は多ステップ
