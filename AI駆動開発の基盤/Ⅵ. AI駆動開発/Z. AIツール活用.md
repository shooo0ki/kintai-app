# Z. AIツール活用

## 概要
AI駆動開発には複数のツールがあります。コード補完、チャットベース開発、AIペアプログラミングそれぞれ異なるシーンで力を発揮します。

---

## 1. コード補完ツール

### 1.1 役割
GitHub Copilotなどの補完ツールは開発中の「小さな助手」です。関数名や変数名を入力すると、実装を自動提案してくれます。

### 1.2 効果的な使い方
既知のパターン（`formatDate(date) {`）やボイラープレート、テストコードで力を発揮します。説明的な関数名、型情報（TypeScript）、コンテキストコメントを提供することで補完品質が向上します。

### 1.3 注意点
```
❌ セキュリティリスク: 脆弱なパターンを提案する場合がある
   password = input.replace(/[^a-z]/g, '');  // 脆弱
✅ 改善: const password = crypto.scryptSync(input, salt, 32);

❌ パフォーマンス問題: 「動く」コード優先で非効率なことがある
✅ 補完後は計算量（O(n) vs O(n²)）を確認
```

---

## 2. チャットベース開発

### 2.1 特徴
Claude、ChatGPTなどのチャット型AIは対話を通じて複雑なコード生成が可能です。補完ツールは小単位（数行）、チャット型は大単位（複数ファイル）で動作します。

### 2.2 効果的なフロー
要件を全体像から説明（「以下の3つの機能を持つ認証システムが必要」）し、段階的に対話します。断片的な要件指示（「ログイン機能作って」「トークン認証追加」）では各機能が独立になるため避けます。

### 2.3 利用シーン
- 複雑なAPI設計
- 全体的なアーキテクチャ
- 新パラダイムの学習

---

## 3. AIペアプログラミング

### 3.1 形式
人間がドライバー＆ナビゲータの役割を担い、AIが補助者として実装をサポートします。人間が常に方向性をコントロールします。

### 3.2 実践的なフロー
```
1. 人間が設計を説明（「POST /api/orders が必要。userId確認、在庫確認、トランザクション内で登録」）
2. AIが枠組みを提案（「1.バリデーション 2.userId確認 3.在庫確認...」）
3. 人間が詳細指示（「2番目のuserIdチェックは404エラー返す」）
4. AIが実装提案
5. 人間が確認・次の指示
6. 繰り返し完成度を上げる
```

### 3.3 利点と課題
```
利点：人間が方向性をコントロール、段階的評価、学習効果
課題：スピード低下（会話オーバーヘッド）、検証手間

対策：定型的なタスクは補完ツール、複雑な設計はペアプロ、時間がない場合は全てAI任せで後検証
```

---

## 4. ツール選択ガイド

| タスク | 推奨ツール | 理由 |
|--------|-----------|------|
| 関数の数行補完 | 補完ツール | リアルタイム |
| テストコード生成 | 補完ツール | パターン明確 |
| 複雑なAPI設計 | チャット型AI | 対話が必要 |
| アーキテクチャ | チャット型AI | 複数ファイル跨ぎ |
| バグ修正相談 | ペアプロ | 人間の関与必要 |
| CRUD実装 | 補完ツール | パターン化済み |

---

## まとめ

**重点項目:**
- 補完ツール：小単位の効率化
- チャット型AI：複雑な要件対応
- ペアプロ：判断が必要な部分
- 臨機応変に組み合わせて活用
