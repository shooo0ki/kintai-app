# F. プログラミングパラダイム

## 概要

プログラムを組み立てる根本的なアプローチです。関数型・手続き型・オブジェクト指向・宣言型などがあります。AI駆動開発では、複数のパラダイムを理解し、最適なパターンを選択する判断力が必要です。

---

## 1. 関数型プログラミング

**不変性（immutability）：** データを変更せず、新しいデータを返す。同じ入力に対して常に同じ出力になります。

**純粋関数（pure function）：** 入出力だけで決まり、外部状態に影響しない関数。テスト容易性が高く、AI生成コードのレビューも簡単です。

**高階関数：** 関数を引数として受け取ったり、戻り値として返したりする関数。戦略の差し替えや関数の合成に活用します。

**map/filter/reduce：** 配列変換の基本パターン。意図が明確で、手続き型ループより読みやすい。

**副作用の分離：** I/Oやログを境界に集め、内部は純粋関数で構成。計算ロジックがテスト容易で変更が簡単になります。

```javascript
// filter：条件で選別、map：各要素を変換
const result = numbers
  .filter(x => x > 2)
  .map(x => x * 2);
```

---

## 2. 手続き型プログラミング

「どのように実行するか」をステップバイステップで記述します。スクリプト処理やシンプルな処理フローに適しています。

---

## 3. オブジェクト指向プログラミング

データと処理を「オブジェクト」で組織します。**カプセル化**で内部詳細を隠し、**継承**でコード重複を削減します。

```javascript
class User {
  constructor(name, email) {
    this.name = name;
    this.email = email;
  }
  displayInfo() {
    return `${this.name} (${this.email})`;
  }
}
```

---

## 4. 宣言型プログラミング

「何をしたいのか」を記述します。SQLやHTMLが典型例で、処理の意図が明確になります。

---

## 5. 複数パラダイムの組み合わせ

実務では複数のパラダイムを組み合わせます。関数型（filter/map）+ オブジェクト指向（クラス）など、各場面で最適なものを選択することが重要です。

---

## まとめ

**重点項目：** 関数型（不変性・純粋関数）、手続き型（ステップ明確）、オブジェクト指向（カプセル化）、宣言型（意図明確）。適切なパラダイムを選択することで、保守性と拡張性に優れたコードが実現できます。
