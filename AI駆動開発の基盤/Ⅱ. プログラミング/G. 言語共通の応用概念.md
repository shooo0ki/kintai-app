# G. 言語共通の応用概念

## 概要

ジェネリクス、型推論、非同期処理、イテレータなどの概念は、JavaScript・Python・Go・Rustなど言語を問わず共通して現れます。これらを言語横断的に理解することで、新しい言語への学習効率が大幅に向上し、AI駆動開発での適応力が高まります。

---

## 1. ジェネリクス（汎用型）

型そのものを「変数」のように扱い、再利用可能なコンポーネントを作成します。型ごとにコードを書く必要がなくなり、型制約で特定の型だけを受け付けられます。

---

## 2. 型推論

コンパイラ・インタプリタが文脈から型を自動推測します。明示的に書かなくても型チェックが機能し、推論が曖昧な場合のみ明示的に指定します。

---

## 3. 非同期処理

I/O・ネットワーク・タイマーなど、時間がかかる操作を効率的に処理します。**Promise** で非同期処理の結果を管理し、**async/await** で簡潔に記述。複数の非同期処理は **Promise.all** で同時実行します。

```javascript
async function fetchUserData(userId) {
  const userResponse = await fetch(`/api/users/${userId}`);
  const user = await userResponse.json();
  return user;
}
```

---

## 4. イテレータとジェネレータ

**イテレータ：** 要素を1つずつ順番に処理する仕組み。for...of で利用されます。

**ジェネレータ関数：** yield で値を1つずつ返す関数。メモリ効率的で遅延評価が可能。無限シーケンス表現や処理の流れが明確になります。

```javascript
function* countUp(n) {
  for (let i = 1; i <= n; i++) {
    yield i;
  }
}
```

---

## 5. ポリモーフィズム（多態性）

**メソッドオーバーライド：** 基本クラスのメソッドを派生クラスで上書きして、同じインターフェースで異なる動作を実現。

**ダックタイピング：** 「同じメソッドを持つ」=「同じ型」として扱う柔軟なアプローチ。型の厳密性より実装の柔軟性を優先します。

---

## 6. メモ化とキャッシング

計算結果をキャッシュして、同じ計算を避け、パフォーマンスを向上させます。

---

## まとめ

**重点項目：** ジェネリクス（型の再利用）、型推論（柔軟な型チェック）、非同期処理（効率的な待機）、イテレータ（遅延評価）、ポリモーフィズム（柔軟な実装）、メモ化（パフォーマンス最適化）。言語横断的に理解することで、新しい言語への適応が容易になります。
