# J. 開発環境構築

## 概要

開発環境構築は、ターミナル操作・エディタ・ランタイム管理・環境変数設定を組み合わせた基盤です。パスの概念（相対/絶対）を理解し、標準出力と標準エラーを区別してリダイレクトする。エディタのLSP統合で定義ジャンプや安全なリネームが可能になり、nvm や pyenv でランタイムバージョンを切り替える。.env ファイルで秘密情報を管理し、.env.example でチーム共有する。これらが揃うことで日々の開発が効率化されます。

### ターミナル操作

パスの概念が中心。相対パス（../ 、./ ）と絶対パス（/usr/local など）を使い分け、cd で移動、ls で確認。出力制御はリダイレクト（> 、2>&1）で実行ログを保存し、grep や tail で条件検索する。

```bash
# パス操作とリダイレクト
cd /home/user/projects/myapp
npm install > install.log 2>&1

# パイプで段階的に絞り込み
ps aux | grep node | grep -v grep
```

### エディタ/IDEとランタイム管理

IDEの言語サーバ統合で定義ジャンプ・参照検索・リネーム機能が使える。保存時自動フォーマット（prettier）と lint 修正（eslint --fix）は setting.json で共有設定。デバッガはブレークポイント・ステップ実行・ウォッチで複雑なバグを追跡。

ランタイムはバージョンマネージャで管理。nvm（Node.js）や pyenv（Python）で複数バージョンを切り替え、.nvmrc や .python-version でチーム統一。PATH の順序が重要—先に見つかったバージョンが優先される。

```bash
# ランタイムバージョン管理
nvm install 18.16.0
nvm use 18.16.0
which node  # 使用中の実行ファイル確認
```

### 環境変数と秘密管理

.env（Git無視）に秘密情報を記述し、.env.example（Git管理）で構造を共有。起動時に環境変数の存在とバリデーションをチェックし、不正な値ならエラー停止。本番では CI/CD で環境変数を注入し、開発環境より高いセキュリティレベルで管理する。
