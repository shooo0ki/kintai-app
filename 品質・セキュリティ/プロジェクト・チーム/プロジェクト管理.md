# プロジェクト管理

## 概要
スクラム、カンバンを活用してタスク可視化、進捗追跡、チーム同期を効率化し、予測可能で柔軟な運営を実現します。

## スクラム（時間ボックス化された反復）

**サイクル**：スプリント計画（4時間） → 日次スタンドアップ（15分/日） → 開発・テスト・レビュー（10営業日） → スプリントレビュー（2時間） → 振り返り（1.5時間）

**メトリクス**：ストーリーポイント（見積もり単位）、ベロシティ（1スプリントの完了ポイント）、バーンダウン（残ポイント推移）

```python
class SprintManagement:
    def __init__(self, sprint_duration_days=14):
        self.sprint_backlog = []
        self.burndown = []

    def plan_sprint(self, selected_items, sprint_goal):
        self.sprint_goal = sprint_goal
        for item in selected_items:
            self.sprint_backlog.append({
                'title': item['title'],
                'story_points': item['estimated_points'],
                'status': 'backlog'
            })

    def calculate_sprint_metrics(self):
        completed_points = sum(
            task['story_points'] for task in self.sprint_backlog
            if task['status'] == 'done'
        )
        total_points = sum(task['story_points'] for task in self.sprint_backlog)
        return {
            'velocity': completed_points,
            'completion_rate': f"{(completed_points / total_points * 100):.1f}%"
        }
```

## カンバン（継続的フロー）

固定スプリント不要。Backlog → TODO（WIP:5） → In Progress（WIP:3） → Review（WIP:2） → Done

WIP制限で並列度制御、ボトルネック可視化、サイクルタイム短縮。

```python
class KanbanBoard:
    def __init__(self):
        self.columns = {
            'backlog': {'max_wip': float('inf')},
            'todo': {'max_wip': 5},
            'in_progress': {'max_wip': 3},
            'review': {'max_wip': 2},
            'done': {'max_wip': float('inf')}
        }

    def move_task(self, task_id, to_column):
        if len(self.columns[to_column]['tasks']) < self.columns[to_column]['max_wip']:
            return True
        return False

    def detect_bottlenecks(self):
        return [col for col, data in self.columns.items()
                if len(data['tasks']) / data['max_wip'] > 0.8]
```

## スプリント計画のベストプラクティス

- **見積もり手法**：Planning Poker（独立見積もり → 議論 → 合意）で精度向上
- **予測可能性**：複数スプリント実績から平均ベロシティ算出、次期計画に反映
- **優先度固定**：スプリント中の大幅変更を避け、フォーカス維持
- **障害枠**：全容量の10～20%を緊急対応用に確保
- **透明性**：ステークホルダーが常にプロジェクト状況を把握
- **フィードバックループ**：顧客・市場からの声を次スプリントに反映
